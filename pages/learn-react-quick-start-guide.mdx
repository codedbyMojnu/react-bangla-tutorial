## 1. পরিচিতি ও লক্ষ্য

এই টিউটোরিয়ালের লক্ষ্য — **React শেখানো এমনভাবে যাতে ৮০% ব্যবহারিক কাজ আপনি এই টিউটোরিয়াল পড়ে করে ফেলতে পারেন।** আপনি যদি প্রতিটি অংশ মনোযোগ দিয়ে পড়েন এবং অনুশীলন করেন, তাহলে আপনি ছোট ও মাঝারি সাইজ প্রজেক্ট বানাতে সক্ষম হবেন।

## 2. প্রি-রেকুইজিট ও সেটআপ

### প্রি-রেকুইজিট

* HTML, CSS (মৌলিক ধারণা)
* আধুনিক JavaScript (ES6+) — `let/const`, arrow functions, template strings, destructuring, modules

### কম্পিউটার সেটআপ

1. Node.js ও npm (বা pnpm/yarn) ইন্সটল করুন
2. কোড এডিটর: VS Code (ইনস্টল)

```bash
cd react-bangla

npm run dev
```

> Vite দ্রুত, হট-রিলোড আছে — তাই Vite দিয়ে রিয়েক্ট প্রোজেক্ট শুরু করা ভালো।

## 3. আপনার প্রথম React অ্যাপ্লিকেশন: প্রোজেক্ট ফাইল ও ফোল্ডার স্ট্রাকচার

বর্তমানে মানুষজন প্রোজেক্ট শুরু করে `Scaffolding` দিয়ে, এটা হলো-> আগে ফোল্ডার স্ট্রাকচার এবং প্রয়োজনীর কিছু প্যাকেজ, কম্পোনেন্টস দিয়ে তৈরি করা প্রোজেক্ট। যেটা ডেভেলপারদের কাজ অনেক সহজ করে, ডেভেলপার দ্রুত বুঝতে পারে-> এখন তার কিভাবে আগানো উচিত? সবকিছু মনে পড়ে যায়। কারণ- প্রত্যেকটা প্রোজেক্টের সিস্টেম তো একই। 

```
react-bangla/
  ├─ index.html
  ├─ src/
  │   ├─ main.jsx
  │   ├─ App.jsx
  │   ├─ components/
  │   │   ├─ Header.jsx
  │   │   └─ Button.jsx
  │   └─ styles.css
  └─ package.json
```

`main.jsx` থেকে `App` রেন্ডার করে: এখানে `ReactDOM.createRoot(...).render(<App />)` থাকবে।

## 4. JSX — HTML-এর মত কিন্তু JS এর ভিতর

JSX হলো JavaScript এর ভিতরে HTML-এর মত সিনট্যাক্স। এখন আমরা একটি Welcome কম্পোনেন্ট বানাবো। 

**মনে রাখার নিয়ম**

* JSX-এ attribute `className` ব্যবহার করতে হবে।
* একাধিক ট্যাগ হলে এক parent লাগবে (বা `<>...</>` fragment)।
* JavaScript expression `{}` দিয়ে লিখতে হবে। 

```jsx
function Welcome({ name }) {
  return (
    <div className="welcome">
      <h1>স্বাগতম, {name}!</h1>
    </div>
  );
}
```

## 5. React কম্পোনেন্ট কি— ছোট ছোট UI ব্লক 

React-এ প্রতিটি UI অংশই একটা কম্পোনেন্ট। ফাংশনাল কম্পোনেন্ট সবচেয়ে জনপ্রিয়।

```jsx
export default function Header() {
  return <header><h1>React-Bangla</h1></header>;
}
```

কোম্পোনেন্ট রিইউজ করা সহজ — `<Header />` একাধিক ফাইলে ব্যবহার করুন।

ঠিক আছে। ছোট করে ক্লাস কম্পোনেন্ট ব্যাখ্যা করলে এভাবে হবে:

---

### React ক্লাস কম্পোনেন্ট

React-এ কম্পোনেন্ট দুইভাবে বানানো যায়: **ফাংশনাল** আর **ক্লাস**। ক্লাস কম্পোনেন্ট পুরনো স্টাইল, কিন্তু এখনও কাজ করে।

**উদাহরণ:**

```jsx
import React, { Component } from "react";

export default class Header extends Component {
  render() {
    return <header><h1>React-Bangla</h1></header>;
  }
}
```

**কী জিনিস:**

1. `class Header extends Component` → কম্পোনেন্ট তৈরি করা হলো।
2. `render()` → এখানে UI JSX রিটার্ন হয়।
3. `<Header />` → যেকোনো জায়গায় ব্যবহার করা যায়।

**সংক্ষেপে:**

* ক্লাস কম্পোনেন্ট হলো **পুরনো স্টাইলের React কম্পোনেন্ট**, যেটাতে `render()` মেথড থাকে।
* ফাংশনাল কম্পোনেন্টের মতো একাধিক বার রিইউজ করা যায়।

## 6. Props — ডাটা পাস করা

Props হলো কম্পোনেন্টে বাইরের ডাটা পাঠানোর উপায়।

```jsx
function Greeting({ name }) {
  return <p>হ্যালো, {name}!</p>;
}

// ব্যবহার
<Greeting name="Mojnu" />
```

## 7. State ও useState — UI কে জীবন্ত করা

State হলো লোকাল ডাটা যা কম্পোনেন্ট লাইফটাইম ধরে রাখে।

```jsx
import { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);
  return (
    <div>
      <p>সংখ্য়া: {count}</p>
      <button onClick={() => setCount(count + 1)}>বড়াও</button>
    </div>
  );
}
```

কিছু টিপস:

* `setCount` asynchronous হতে পারে — যদি নতুন মান পুরোনোর ওপর নির্ভর করে, `setCount(c => c + 1)` ব্যবহার করুন।
* প্রতিটি কম্পোনেন্টের state আলাদা।

## 8. ইভেন্ট ও ইউজার ইন্টারঅ্যাকশন

ইভেন্ট হ্যান্ডলার পাস করুন যেমন `onClick`, `onChange`।

```jsx
function MyButton() {
  function handleClick() { alert('আপনি ক্লিক করেছেন!'); }
  return <button onClick={handleClick}>ক্লিক করো</button>;
}
```

মনে রাখবেন `onClick={handleClick}` — না `handleClick()`।

## 9. লিস্ট, key ও রেন্ডার অপ্টিমাইজেশন

**List render**

```jsx
const items = [{id:1, text:'A'}, {id:2, text:'B'}];
return <ul>{items.map(i => <li key={i.id}>{i.text}</li>)}</ul>;
```

Key দিন যাতে React প্রতিটি আইটেম ট্র্যাক করতে পারে।

## 11. ফর্ম ও ইনপুট হ্যান্ডলিং

Controlled component প্যাটার্ন:

```jsx
function NameForm() {
  const [name, setName] = useState('');
  return (
    <form onSubmit={(e)=>{e.preventDefault(); alert(name);}}>
      <input value={name} onChange={e => setName(e.target.value)} />
      <button type="submit">সাবমিট</button>
    </form>
  );
}
```

## 12. useEffect — সাইড-ইফেক্ট ও ডেটা ফেচিং

`useEffect` ব্যবহার করে API কল বা সাবস্ক্রিপশন করা হয়।

```jsx
import { useEffect, useState } from 'react';

function Users() {
  const [users, setUsers] = useState([]);
  useEffect(() => {
    fetch('https://jsonplaceholder.typicode.com/users')
      .then(r => r.json())
      .then(setUsers);
  }, []); // খালি ডিপেন্ডেন্সি → মাউন্টের সময় একবার

  return <ul>{users.map(u => <li key={u.id}>{u.name}</li>)}</ul>;
}
```

**Cleanup:** সাবস্ক্রিপশন বা টাইমার থাকলে `return` করে ক্লিনআপ লিখুন।

## 13. কম্পোনেন্টদের মধ্যে ডাটা শেয়ার

### Lifting state up

যদি দুই চাইল্ড একই স্টেট শেয়ার করে, প্যারেন্টে state রাখুন ও props দিয়ে পাঠান।

### Context API

কমনভাবে ব্যবহার্য থিম বা ইউজার অবজেক্ট শেয়ার করা হলে `React.createContext()` ব্যবহার করুন।

## 14. রিয়েক্ট রাউটিং (React Router)

React Router দিয়ে মাল্টিপেজ UI বানান। সংক্ষিপ্ত উদাহরণ:

```jsx
import { BrowserRouter, Routes, Route } from 'react-router-dom';

<BrowserRouter>
  <Routes>
    <Route path="/" element={<Home />} />
    <Route path="/about" element={<About />} />
  </Routes>
</BrowserRouter>
```

## 15. কাস্টম হুক — reusable logic

যদি অনেক কম্পোনেন্টে একই লজিক লাগে, কাস্টম হুক বানান:

```jsx
import { useState, useEffect } from 'react';

export function useFetch(url) {
  const [data, setData] = useState(null);
  useEffect(() => { fetch(url).then(r=>r.json()).then(setData); }, [url]);
  return data;
}
```

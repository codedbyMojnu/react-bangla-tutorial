## ফোল্ডার এবং ফাইলের অর্গানাইজের নিয়মাবলী

### টপ-লেভেল ফোল্ডার

টপ-লেভেল ফোল্ডারগুলো আপনার অ্যাপ্লিকেশনের কোড এবং স্ট্যাটিক অ্যাসেট (যেমন ছবি বা ফন্ট) গুছিয়ে রাখার জন্য ব্যবহৃত হয়।

| ফোল্ডার      | বর্ণনা                                      |
| :----------- | :------------------------------------------ |
| **`app`**    | App Router ব্যবহারের জন্য।                  |
| **`pages`**  | Pages Router ব্যবহারের জন্য।                |
| **`public`** | স্ট্যাটিক ফাইল (যেমন ছবি, আইকন) রাখার জন্য। |
| **`src`**    | অ্যাপ্লিকেশনের মূল কোড রাখার জন্য (ঐচ্ছিক)। |

---

### টপ-লেভেল ফাইল

টপ-লেভেল ফাইলগুলো অ্যাপ্লিকেশন কনফিগারেশন, প্যাকেজ ম্যানেজমেন্ট এবং পরিবেশগত ভেরিয়েবল সেট করার জন্য ব্যবহৃত হয়।

| ফাইল                 | বর্ণনা                                                      |
| :------------------- | :---------------------------------------------------------- |
| **`next.config.js`** | Next.js-এর মূল কনফিগারেশন ফাইল।                             |
| **`package.json`**   | প্রজেক্টের নির্ভরতা (dependencies) এবং স্ক্রিপ্ট।           |
| **`middleware.ts`**  | অনুরোধ (request) পরিচালনার জন্য মিডলওয়্যার।                |
| **`.env`**           | পরিবেশগত ভেরিয়েবল (Environment variables) সেট করার জন্য।   |
| **`.gitignore`**     | Git দ্বারা ট্র্যাক করা হবে না এমন ফাইল বা ফোল্ডারের তালিকা। |
| **`tsconfig.json`**  | TypeScript-এর জন্য কনফিগারেশন ফাইল।                         |

---

### রাউটিং ফাইল

এই ফাইলগুলো আপনার অ্যাপ্লিকেশনের বিভিন্ন পৃষ্ঠা এবং লেআউট নির্ধারণ করে।

| ফাইল               | বর্ণনা                                           |
| :----------------- | :----------------------------------------------- |
| **`layout.js`**    | একাধিক পৃষ্ঠার জন্য একটি সাধারণ লেআউট তৈরি করে।  |
| **`page.js`**      | একটি নির্দিষ্ট রুটের জন্য ইউনিক UI তৈরি করে।     |
| **`loading.js`**   | পৃষ্ঠা লোড হওয়ার সময় একটি লোডিং ইন্টারফেস দেখায়। |
| **`not-found.js`** | কোনো পৃষ্ঠা খুঁজে না পাওয়া গেলে এই UI দেখানো হয়। |
| **`error.js`**     | কোনো ত্রুটি ঘটলে এই UI দেখানো হয়।                |
| **`route.js`**     | সার্ভার-সাইড API এন্ডপয়েন্ট তৈরি করার জন্য।     |

---

### রুট তৈরির বিভিন্ন কৌশল

Next.js-এ বিভিন্ন ধরণের রুট তৈরির সুযোগ রয়েছে।

| রুটের ধরণ        | উদাহরণ          | বর্ণনা                                         |
| :--------------- | :-------------- | :--------------------------------------------- |
| **নেস্টেড রুট**  | `folder/folder` | একটি রুটের ভেতরে আরেকটি রুট তৈরি করা।          |
| **ডাইনামিক রুট** | `[folder]`      | পরিবর্তনশীল ডেটার উপর ভিত্তি করে রুট তৈরি করা। |

---

### রুট গ্রুপিং এবং প্রাইভেট ফোল্ডার

কোডকে আরও সুসংগঠিত করার জন্য এই কৌশলগুলো অত্যন্ত কার্যকর।

- **রুট গ্রুপ `(folder)`**: এই ফোল্ডারগুলো URL পাথে অন্তর্ভুক্ত হয় না। এটি শুধুমাত্র কোডকে যৌক্তিকভাবে সাজানোর জন্য ব্যবহৃত হয়। যেমন, `(marketing)` এবং `(shop)` নামক দুটি আলাদা গ্রুপে ভিন্ন লেআউট ব্যবহার করা যেতে পারে।
- **প্রাইভেট ফোল্ডার `_folder`**: একটি ফোল্ডারের নামের আগে আন্ডারস্কোর (\_) ব্যবহার করলে সেই ফোল্ডার এবং তার ভেতরের কোনো ফাইল রাউটিং সিস্টেমের অংশ হবে না। এটি UI কম্পোনেন্ট বা সহায়ক ফাংশন রাখার জন্য একটি চমৎকার উপায়।

---

### প্রজেক্ট সাজানোর কৌশল

আপনার প্রজেক্টের প্রয়োজন অনুযায়ী একটি কাঠামো বেছে নিতে পারেন।

- **অ্যাপের বাইরে ফাইল রাখা**: `components` বা `lib`-এর মতো ফোল্ডারগুলো `app` ডিরেক্টরির বাইরে রাখা, যাতে রাউটিং এবং মূল কোড আলাদা থাকে।
- **অ্যাপের ভেতরে ফাইল রাখা**: সমস্ত সম্পর্কিত কোড `app` ডিরেক্টরির ভেতরেই রাখা, যাতে সবকিছু এক জায়গায় থাকে।
- **ফিচার বা রুট অনুযায়ী ভাগ করা**: প্রতিটি রুটের জন্য প্রয়োজনীয় কম্পোনেন্ট বা ফাইল সেই রুটের ফোল্ডারের ভেতরেই রাখা।

---

### বিশেষ ব্যবহার

- **একাধিক রুট লেআউট**: অ্যাপ্লিকেশনটিকে সম্পূর্ণ ভিন্ন UI সহ বিভিন্ন বিভাগে ভাগ করার জন্য, প্রতিটি রুট গ্রুপের ভেতরে আলাদা `layout.js` ফাইল তৈরি করুন। মনে রাখবেন, প্রতিটি রুট লেআউটে `<html>` এবং `<body>` ট্যাগ থাকা আবশ্যক।
- **নির্দিষ্ট রুটের জন্য লোডিং স্কেলেটন**: একটি নির্দিষ্ট রুটের জন্য `loading.js` ফাইল প্রয়োগ করতে, রুট গ্রুপ ব্যবহার করে সেই রুটটিকে আলাদা করুন। এতে শুধুমাত্র ওই নির্দিষ্ট রুটের জন্য লোডিং UI দেখানো হবে।

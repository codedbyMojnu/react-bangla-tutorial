# লারাভেল ইন্সটলেশন

এই অধ্যায়ে, আমরা আপনার লোকাল মেশিনে একটি নতুন লারাভেল প্রজেক্ট কিভাবে ইন্সটল এবং রান করতে হয় তা ধাপে ধাপে দেখব।

## লারাভেল পরিচিতি

লারাভেল একটি ওয়েব অ্যাপ্লিকেশন ফ্রেমওয়ার্ক যা অত্যন্ত আকর্ষণীয় এবং সহজ সিনট্যাক্স ব্যবহার করে। একটি ওয়েব ফ্রেমওয়ার্ক আপনার অ্যাপ্লিকেশন তৈরির জন্য একটি কাঠামো এবং সূচনা পয়েন্ট সরবরাহ করে, যা আপনাকে অসাধারণ কিছু তৈরিতে মনোযোগ দিতে সাহায্য করে, আর বাকি সব বিস্তারিত বিষয়গুলো লারাভেল সামলে নেয়।

লারাভেল ডেভেলপারদের একটি অসাধারণ অভিজ্ঞতা দেওয়ার চেষ্টা করে এবং এর পাশাপাশি শক্তিশালী ফিচার যেমন - dependency injection, একটি শক্তিশালী ডাটাবেস অ্যাবস্ট্রাকশন লেয়ার, কিউ এবং নির্ধারিত কাজ (queues and scheduled jobs), ইউনিট এবং ইন্টিগ্রেশন টেস্টিং, এবং আরও অনেক কিছু সরবরাহ করে।

আপনি পিএইচপি ওয়েব ফ্রেমওয়ার্কে নতুন হন বা আপনার বছরের পর বছর অভিজ্ঞতা থাকুক না কেন, লারাভেল এমন একটি ফ্রেমওয়ার্ক যা আপনার সাথে বেড়ে উঠতে পারে।

### কেন লারাভেল?

ওয়েব অ্যাপ্লিকেশন তৈরির জন্য বিভিন্ন সরঞ্জাম এবং ফ্রেমওয়ার্ক রয়েছে। তবে, আমরা বিশ্বাস করি আধুনিক, ফুল-স্ট্যাক ওয়েব অ্যাপ্লিকেশন তৈরির জন্য লারাভেল সেরা পছন্দ।

#### একটি প্রগতিশীল ফ্রেমওয়ার্ক (A Progressive Framework)

আমরা লারাভেলকে একটি "প্রগতিশীল" ফ্রেমওয়ার্ক বলতে পছন্দ করি। এর মানে হলো, লারাভেল আপনার সাথে সাথে বিকশিত হয়। আপনি যদি ওয়েব ডেভেলপমেন্টে প্রথম পা রাখেন, লারাভেলের বিশাল ডকুমেন্টেশন, গাইড এবং ভিডিও টিউটোরিয়াল আপনাকে অভিভূত না করে সবকিছু শিখতে সাহায্য করবে। আর সিনিয়র ডেভেলপারদের জন্য, লারাভেল dependency injection, unit testing, queues, real-time events এর মতো শক্তিশালী টুল সরবরাহ করে।

#### একটি পরিমাপযোগ্য ফ্রেমওয়ার্ক (A Scalable Framework)

লারাভেল অবিশ্বাস্যভাবে পরিমাপযোগ্য (scalable)। পিএইচপি-এর স্কেলিং-বান্ধব প্রকৃতি এবং Redis-এর মতো দ্রুত, ডিস্ট্রিবিউটেড ক্যাশ সিস্টেমের জন্য লারাভেলের বিল্ট-ইন সমর্থনের জন্য ধন্যবাদ, লারাভেলের সাথে হরাইজন্টাল স্কেলিং করা খুবই সহজ। লারাভেল অ্যাপ্লিকেশনগুলো প্রতি মাসে শত শত মিলিয়ন অনুরোধ সহজেই পরিচালনা করার জন্য স্কেল করা হয়েছে।

#### একটি কমিউনিটি ফ্রেমওয়ার্ক (A Community Framework)

লারাভেল পিএইচপি ইকোসিস্টেমের সেরা প্যাকেজগুলোকে একত্রিত করে সবচেয়ে শক্তিশালী এবং ডেভেলপার-বান্ধব ফ্রেমওয়ার্ক সরবরাহ করে। এছাড়াও, বিশ্বজুড়ে হাজার হাজার প্রতিভাবান ডেভেলপার এই ফ্রেমওয়ার্কে অবদান রেখেছেন। কে জানে, হয়তো আপনিও একজন লারাভেল কন্ট্রিবিউটর হয়ে উঠবেন।

## একটি লারাভেল অ্যাপ্লিকেশন তৈরি করা

আপনার প্রথম লারাভেল অ্যাপ্লিকেশন তৈরি করার আগে, নিশ্চিত করুন যে আপনার লোকাল মেশিনে [PHP](https://php.net), [Composer](https://getcomposer.org), এবং [Laravel installer](https://github.com/laravel/installer) ইনস্টল করা আছে। এছাড়াও, আপনার অ্যাপ্লিকেশনের ফ্রন্টএন্ড অ্যাসেট কম্পাইল করার জন্য [Node এবং NPM](https://nodejs.org) অথবা [Bun](https://bun.sh/) ইনস্টল করা উচিত।

### পিএইচপি এবং লারাভেল ইন্সটলার ইন্সটল করা

যদি আপনার লোকাল মেশিনে পিএইচপি এবং কম্পোজার ইনস্টল করা না থাকে, তবে নিচের কমান্ডগুলো macOS, Windows, বা Linux-এ পিএইচপি, কম্পোজার এবং লারাভেল ইন্সটলার ইনস্টল করবে:

**macOS:**

```shell
/bin/bash -c "$(curl -fsSL https://php.new/install/mac/8.4)"
```

**Windows PowerShell (Run as administrator):**

```shell
Set-ExecutionPolicy Bypass -Scope Process -Force; [System.Net.ServicePointManager]::SecurityProtocol = [System.Net.ServicePointManager]::SecurityProtocol -bor 3072; iex ((New-Object System.Net.WebClient).DownloadString('https://php.new/install/windows/8.4'))
```

**Linux:**

```shell
/bin/bash -c "$(curl -fsSL https://php.new/install/linux/8.4)"
```

উপরের কমান্ডগুলো চালানোর পর, আপনার টার্মিনাল সেশন রিস্টার্ট করুন।

যদি আপনার আগে থেকেই পিএইচপি এবং কম্পোজার ইনস্টল করা থাকে, তবে আপনি কম্পোজারের মাধ্যমে লারাভেল ইন্সটলার ইনস্টল করতে পারেন:

```shell
composer global require laravel/installer
```

> **Laravel Herd:** একটি সম্পূর্ণ গ্রাফিকাল পিএইচপি ইনস্টলেশন এবং পরিচালনার অভিজ্ঞতার জন্য, [Laravel Herd](#installation-using-herd) ব্যবহার করতে পারেন।

### অ্যাপ্লিকেশন তৈরি করা

পিএইচপি, কম্পোজার এবং লারাভেল ইন্সটলার ইনস্টল করার পরে, আপনি একটি নতুন লারাভেল অ্যাপ্লিকেশন তৈরি করতে প্রস্তুত। `laravel new` কমান্ডটি আপনার পছন্দের টেস্টিং ফ্রেমওয়ার্ক, ডাটাবেস এবং স্টার্টার কিট নির্বাচন করতে বলবে:

```shell
laravel new example-app
```

অ্যাপ্লিকেশনটি তৈরি হয়ে গেলে, আপনি `dev` কম্পোজার স্ক্রিপ্ট ব্যবহার করে লারাভেলের লোকাল ডেভেলপমেন্ট সার্ভার, কিউ ওয়ার্কার এবং Vite ডেভেলপমেন্ট সার্ভার শুরু করতে পারেন:

```shell
cd example-app
npm install && npm run build
composer run dev
```

ডেভেলপমেন্ট সার্ভার শুরু করার পরে, আপনার অ্যাপ্লিকেশনটি ওয়েব ব্রাউজারে `http://localhost:8000` এ অ্যাক্সেসযোগ্য হবে।

## প্রাথমিক কনফিগারেশন

লারাভেল ফ্রেমওয়ার্কের সমস্ত কনফিগারেশন ফাইল `config` ডিরেক্টরিতে সংরক্ষণ করা হয়। প্রতিটি অপশন ডকুমেন্টেড, তাই ফাইলগুলো দেখুন এবং আপনার জন্য উপলব্ধ অপশনগুলোর সাথে পরিচিত হন।

### এনভায়রনমেন্ট ভিত্তিক কনফিগারেশন

যেহেতু লারাভেলের অনেক কনফিগারেশন অপশনের মান আপনার অ্যাপ্লিকেশনটি লোকাল মেশিনে চলছে নাকি প্রোডাকশন ওয়েব সার্ভারে চলছে তার উপর নির্ভর করে পরিবর্তিত হতে পারে, তাই অনেক গুরুত্বপূর্ণ কনফিগারেশন মান আপনার অ্যাপ্লিকেশনের রুটে থাকা `.env` ফাইল ব্যবহার করে সংজ্ঞায়িত করা হয়।

আপনার `.env` ফাইলটি আপনার অ্যাপ্লিকেশনের সোর্স কন্ট্রোলে কমিট করা উচিত নয়, কারণ আপনার অ্যাপ্লিকেশন ব্যবহারকারী প্রতিটি ডেভেলপার/সার্ভারের জন্য একটি ভিন্ন এনভায়রনমেন্ট কনফিগারেশন প্রয়োজন হতে পারে।

### ডাটাবেস এবং মাইগ্রেশন

এখন যেহেতু আপনি আপনার লারাভেল অ্যাপ্লিকেশন তৈরি করেছেন, আপনি সম্ভবত একটি ডাটাবেসে কিছু ডেটা সংরক্ষণ করতে চাইবেন। ডিফল্টরূপে, আপনার অ্যাপ্লিকেশনের `.env` কনফিগারেশন ফাইলটি নির্দিষ্ট করে যে লারাভেল একটি SQLite ডাটাবেসের সাথে ইন্টারঅ্যাক্ট করবে।

অ্যাপ্লিকেশন তৈরির সময়, লারাভেল আপনার জন্য একটি `database/database.sqlite` ফাইল তৈরি করেছে এবং অ্যাপ্লিকেশনের ডাটাবেস টেবিল তৈরি করার জন্য প্রয়োজনীয় মাইগ্রেশন চালিয়েছে।

আপনি যদি MySQL বা PostgreSQL-এর মতো অন্য ডাটাবেস ড্রাইভার ব্যবহার করতে চান, তবে আপনি আপনার `.env` কনফিগারেশন ফাইলটি আপডেট করতে পারেন। উদাহরণস্বরূপ, আপনি যদি MySQL ব্যবহার করতে চান, আপনার `.env` কনফিগারেশন ফাইলের `DB_*` ভেরিয়েবলগুলো এভাবে আপডেট করুন:

```ini
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=laravel
DB_USERNAME=root
DB_PASSWORD=
```

SQLite ছাড়া অন্য কোনো ডাটাবেস ব্যবহার করতে চাইলে, আপনাকে ডাটাবেস তৈরি করতে হবে এবং আপনার অ্যাপ্লিকেশনের ডাটাবেস মাইগ্রেশন চালাতে হবে:

```shell
php artisan migrate
```

## পরবর্তী ধাপ

এখন যেহেতু আপনি আপনার লারাভেল অ্যাপ্লিকেশন তৈরি করেছেন, আপনি হয়তো ভাবছেন এরপর কী শিখবেন। প্রথমে, আমরা নিম্নলিখিত ডকুমেন্টেশনগুলো পড়ে লারাভেল কীভাবে কাজ করে তার সাথে পরিচিত হওয়ার জন্য দৃঢ়ভাবে সুপারিশ করছি:

- [Request Lifecycle](/docs/12.x/lifecycle)
- [Configuration](/docs/12.x/configuration)
- [Directory Structure](/docs/12.x/structure)
- [Service Container](/docs/12.x/container)
- [Facades](/docs/12.x/facades)

আপনি কিভাবে লারাভেল ব্যবহার করতে চান তা আপনার যাত্রার পরবর্তী পদক্ষেপগুলো নির্ধারণ করবে। আপনি লারাভেলকে একটি ফুল-স্ট্যাক ফ্রেমওয়ার্ক হিসাবে অথবা শুধুমাত্র একটি API ব্যাকএন্ড হিসাবে ব্যবহার করতে পারেন।
